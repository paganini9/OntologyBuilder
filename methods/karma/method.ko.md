# KARMA — 다중 에이전트 LLM 기반 KG 증축

> 출처: Lu, Wu, Zhao, Peng, Wang, *KARMA: Leveraging Multi-Agent LLMs for Automated Knowledge Graph Enrichment*, NeurIPS 2025 Spotlight.

## 1. 한 줄 요약
**여러 전담 에이전트**(엔티티 발견·관계 추출·스키마 정렬·충돌 해소 등)가 분업해, 비정형 텍스트를 반복 파싱·검증·통합하며 **기존 KG를 자동으로 증축(enrichment)** 한다.

## 2. 핵심 개념
- **증축(enrichment)**: 처음부터 짓는 게 아니라 **이미 있는 KG에 새 지식을 더한다**.
- **에이전트 분업**: 핵심 역할 — (a) **엔티티 발견**, (b) **관계 추출**, (c) **스키마 정렬**(기존 클래스/속성과 일치시키기), (d) **충돌 해소**(모순 엣지 제거/병합). 원논문은 9개 협업 에이전트로 PubMed에서 38,230 신규 엔티티, 83.1% 검증 정확도, 충돌 엣지 -18.6% 달성.
- **반복 검증**: 각 에이전트가 산출을 다음 에이전트가 검증/정련한다.

## 3. 구축 프로세스 (단계별)
1. **입력 수집** — `seed_kg.ttl`(기존 KG, 선택)과 `text.txt`(증축에 쓸 비정형 텍스트).
2. **엔티티 발견** — 텍스트에서 새 엔티티 후보를 찾는다.
3. **관계 추출** — 엔티티 쌍 사이의 관계를 추출.
4. **스키마 정렬** — 새 엔티티/관계를 기존 KG의 클래스·속성에 매핑(없으면 신규 생성).
5. **충돌 해소** — 기존 KG와 모순되거나 중복되는 엣지를 정리.
6. **병합·산출** — 단계(발견→추출→정렬→충돌해소)별 스냅샷과 `ontology.ttl`·`ontology.json`·`steps.json`을 쓴다.

## 4. 입력 / 출력
| 구분 | 파일 | 설명 |
|------|------|------|
| 입력 | `text.txt` | 증축에 쓸 비정형 텍스트 |
| 입력 | `seed_kg.ttl` | 기존 KG(선택; 없으면 빈 KG에서 시작) |
| 출력 | `ontology.ttl` | 증축된 OWL (Turtle) |
| 출력 | `ontology.json` | Cytoscape nodes/edges (seed vs 신규 구분) |
| 출력 | `steps.json` | 에이전트 단계별 스냅샷 |

## 5. LLM 백엔드
- 기본 `mock`: 키 없이 결정론적으로 동작. 발견·추출은 cqbycq 휴리스틱, 정렬은 기존 KG 노드와의 이름 매칭(단수/복수·대소문자), 충돌 해소는 중복/역방향 엣지 제거 규칙.
- `gemini`/`anthropic`(api): 키가 있으면 실제 LLM이 각 에이전트 역할을 수행. 키 없으면 자동 MOCK 폴백.

## 6. 직접 해보기
1. `samples/text.txt`(와 선택적으로 `samples/seed_kg.ttl`)를 자신의 자료로 교체.
2. **Run**(또는 `python pipeline.py samples runs/out`) 실행.
3. 단계 슬라이더로 발견→추출→정렬→충돌해소로 KG가 증축되는 과정을 확인.
