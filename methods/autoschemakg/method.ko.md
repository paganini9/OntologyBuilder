# AutoSchemaKG — 동적 스키마 귀납 + 추출

## 한 줄 요약
사전 정의된 스키마 **없이** 원시 텍스트에서 지식 그래프를 만든다: 트리플을 추출하면서 동시에 스키마(엔티티 타입 + 관계 타입)를 **데이터로부터 상향식(bottom-up)으로 귀납**한다.

## 핵심 개념
전통적인 KG/온톨로지 파이프라인은 사람이 미리 작성한 스키마(클래스, 속성)에서 출발해 텍스트를 거기에 맞춘다. AutoSchemaKG (Bai 외, arXiv 2505.23628, 2025)는 이를 뒤집는다: 스키마는 **입력이 아니라 추출의 산출물**이다.

- **스키마를 주지 않는다.** 모델은 먼저 텍스트를 읽고 `(주어, 관계, 목적어)` 트리플을 뽑는다.
- **스키마는 인스턴스로부터 떠오른다.** 추출된 모든 엔티티에 타입을 부여하고, 모든 관계 레이블이 관계 타입이 된다 — 전부 실제 말뭉치에서 발견된 것으로부터 귀납된다.
- 따라서 스키마를 사전에 열거할 수 없는 오픈 도메인·웹 스케일 텍스트에 적합하다.

기억할 핵심: AutoSchemaKG에서 온톨로지/스키마는 **미리 주어지는 것이 아니라 데이터로부터 귀납된다(INDUCED).**

## 구축 프로세스 (단계별)
1. **단계 A — 트리플 추출.** 각 문장을 LLM에 보내 `(주어, 관계, 목적어)` 트리플을 받는다. 주어·목적어는 *인스턴스*(구체적 엔티티)가 되고 관계 레이블은 모인다. 문장마다 스냅샷 단계가 하나씩 생겨 인스턴스 그래프가 자라는 과정을 보여준다.
2. **단계 B — 스키마 귀납(상향식).** 인스턴스가 모이면, 결정론적 귀납 규칙으로 각 인스턴스에 엔티티 타입을 부여한다(목 구현에서는 작은 키워드/접미사 표 → `Device` / `Material` / `System` / `Entity`). 서로 다른 관계 레이블들은 관계 타입이 된다. 귀납된 타입마다 클래스 노드를 추가하고, 각 인스턴스를 자기 타입에 잇는 `instanceOf` 엣지를 만든다. *스키마*가 등장하는 유일한 단계이며 — 처음부터 주어진 적이 없다.
3. **직렬화.** 귀납된 타입은 `owl:Class`, 인스턴스는 자기 클래스로 타입 지정된 `owl:NamedIndividual`, 관계는 `owl:ObjectProperty`가 되고 개체 사이에 트리플이 채워진다.

## 입력·출력

| | 파일 | 설명 |
|---|---|---|
| 입력 | `samples/text.txt` | 자유 텍스트, 한 줄당 한 문장(또는 `.`로 구분). 스키마 없음. |
| 출력 | `ontology.ttl` | 귀납된 스키마 + 인스턴스를 OWL/Turtle로(rdflib). |
| 출력 | `ontology.json` | Cytoscape 그래프: 클래스 노드(귀납된 타입) + 인스턴스 노드 + 관계/`instanceOf` 엣지. |
| 출력 | `steps.json` | 문장마다 스냅샷 한 개(추출) + 마지막 `(schema induction)` 단계. |
| 출력 | `manifest.json` | 백엔드, 카운트(인스턴스 / 귀납 클래스 / 트리플 / 관계). |

## LLM 백엔드
추출 단계(단계 A)만 LLM을 호출하며 `backend.llm.get_backend`를 통한다. `--backend mock`이면 결정론적 `mock_responder`가 문장을 트리플로 파싱하므로 전체 파이프라인이 **API 키 없이** 돌고 재현 가능한 출력을 낸다. 스키마 귀납(단계 B)은 결정론적 후처리라 LLM이 전혀 필요 없다. 키를 설정하면 실제 백엔드(`api`: gemini / anthropic)가 단계 A를 대체할 수 있다.

## 직접 해보기
```bash
python methods/autoschemakg/pipeline.py methods/autoschemakg/samples runs/out --backend mock
python -m pytest methods/autoschemakg/tests/ -q
```
사이트 뷰어에서 `runs/out/ontology.json`을 열면 인스턴스가 먼저 추출되고, 마지막 스키마 귀납 단계에서 귀납된 클래스가 나타나는 과정을 볼 수 있다.
