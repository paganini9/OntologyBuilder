# iText2KG — 증분 zero-shot 지식그래프 구축

> 출처: Lairgi, Moncla, Cazabet, Benabdeslem, Cléau, *iText2KG: Incremental Knowledge Graphs Construction Using Large Language Models*, WISE 2024 (arXiv:2409.03284). 코드: github.com/AuvaLab/itext2kg.

## 1. 한 줄 요약
여러 문서를 **하나씩 증분(incremental)** 으로 처리하며, **문서 정제 → 증분 엔티티 추출 → 증분 관계 추출 → 그래프 통합** 4모듈로, 사전 스키마·후처리 없이 의미 중복을 제거하며 지식그래프를 키운다.

## 2. 핵심 개념
- **증분 구축**: 문서를 한 번에 합치지 않고 **문서 단위로 순차** 처리해 누적 그래프에 더한다. 새 문서의 엔티티/관계는 **이미 있는 것과 의미적으로 대조**해 중복을 만들지 않는다.
- **Document Distiller**: 원문을 추출에 적합한 **의미 블록(semantic blocks)** 으로 정제(요약·정규화)한다.
- **증분 엔티티/관계 추출기**: 정제된 블록에서 엔티티를, 그다음 관계를 추출하되 **전역 엔티티 집합을 참조**해 같은 대상을 하나로 묶는다(zero-shot).
- **Graph Integrator**: 추출 결과를 누적 그래프에 통합하고 시각화한다.

## 3. 구축 프로세스 (단계별)
1. **문서 수집** — `text.txt` 에 여러 문서(빈 줄로 구분). 한 문서가 한 증분 단위.
2. **문서 정제(Distill)** — 각 문서를 의미 블록으로 정제(MOCK은 공백 정규화/문장 정리).
3. **증분 엔티티 추출** — 블록에서 엔티티(대문자 명사)를 뽑고, **누적 엔티티와 의미 매칭**(단수/복수·대소문자)해 중복을 통합.
4. **증분 관계 추출** — 엔티티 쌍 사이 관계(객체 속성)를 추출하고 누적.
5. **그래프 통합** — 새 노드/엣지를 그래프에 합치고 중복 엣지 제거. **문서마다 스냅샷**을 남겨 "문서가 더해질수록 그래프가 커지는" 증분 과정을 보여준다.
6. **산출** — `ontology.ttl`·`ontology.json`·`steps.json`.

## 4. 입력 / 출력
| 구분 | 파일 | 설명 |
|------|------|------|
| 입력 | `text.txt` | 여러 문서(빈 줄로 구분; 한 문서=한 증분 단위) |
| 출력 | `ontology.ttl` | OWL (Turtle) |
| 출력 | `ontology.json` | Cytoscape nodes/edges |
| 출력 | `steps.json` | 문서별 증분 스냅샷 — 단계 재생용 |

## 5. LLM 백엔드
- 기본 `mock`: 키 없이 결정론적으로 동작. 정제는 텍스트 정규화, 엔티티는 대문자 명사 + 누적 의미 중복 제거, 관계는 관계동사 패턴.
- `gemini`/`anthropic`(api): 키가 있으면 실제 LLM이 4모듈을 수행해 더 정교한 정제·추출·중복해소. 키 없으면 자동 MOCK 폴백.

## 6. 직접 해보기
1. `samples/text.txt` 를 자신의 문서 모음(빈 줄로 구분)으로 교체.
2. **Run**(또는 `python pipeline.py samples runs/out`) 실행.
3. 단계 슬라이더로 문서가 하나씩 추가되며 중복 없이 그래프가 자라는 과정을 확인.
