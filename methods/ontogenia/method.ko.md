# Ontogenia — 반복 + 메타인지 프롬프팅으로 CQ를 OWL로

> 출처: Lippolis, Ceriani et al., *Ontology Generation using Large Language Models*, ESWC 2024 (arXiv:2503.05388, Springer DOI 10.1007/978-3-031-78952-6_38) 의 **Ontogenia** 변형.

## 1. 한 줄 요약
**역량 질문(CQ)** 을 처리하되, CQbyCQ와 달리 이전까지 만든 온톨로지를 **맥락으로 들고 가며(memoryful)** LLM이 자기 출력을 **스스로 비평·수정(메타인지 프롬프팅)** 하는 반복 루프로 온톨로지를 정련해 나간다.

## 2. 핵심 개념
- **역량 질문(CQ)**: 온톨로지가 답할 수 있어야 하는 질문. 예) "제품은 어떤 부품들로 구성되는가?"
- **Memoryful (맥락 유지)**: 각 CQ를 처리할 때 **이미 누적된 온톨로지**를 프롬프트 컨텍스트로 함께 준다. 그래서 새 조각이 기존 클래스/속성과 정합되도록 만든다. (CQbyCQ는 CQ를 서로 독립적으로 보는 *memoryless* — 이것이 두 기법의 핵심 차이.)
- **메타인지 프롬프팅(Metacognitive Prompting, MP)**: LLM에게 "방금 만든 온톨로지 조각이 CQ를 제대로 답하는가? 기존 정의와 충돌하거나 중복되는가? 더 적절한 이름·계층은 없는가?" 를 **스스로 점검(self-reflection)** 하게 한 뒤 수정안을 내게 한다.
- **디자인 패턴 가이드**: 온톨로지 디자인 패턴(ODP)을 참조해 조각의 구조(클래스 계층, 속성 domain/range)를 잡도록 유도한다.
- **반복 정련 루프**: 초안 → 자기비평 → 수정 → 병합을 CQ마다 돌고, 마지막에 전체 일관성 정련을 한 번 더 한다.

## 3. 구축 프로세스 (단계별)
1. **CQ 수집** — `competency_questions.txt` 한 줄에 하나씩.
2. **CQ → 조각 초안 (반복, 맥락 포함)** — CQ마다 LLM(또는 MOCK 휴리스틱)에게 **현재까지 누적된 온톨로지**를 함께 주고 JSON 조각 `{classes, object_properties, data_properties, restrictions}` 초안을 만들게 한다.
3. **자기비평 패스(메타인지)** — 같은 CQ에 대해 초안을 다시 LLM에 넣어 self-critique 한다. 이 패스는 (a) 기존 클래스와의 **중복 병합**, (b) 더 정확한 **이름 변경(rename)**, (c) 누락된 **상위/하위 클래스·제약 추가** 를 제안해 **수정된 조각**을 만든다. → 여기서 CQbyCQ에는 없는 추가 단계 스냅샷이 생긴다.
4. **병합** — 수정된 조각을 누적 그래프에 합친다. 속성 domain/range로 참조된 클래스는 자동 생성, 중복은 기존 정의로 통합.
5. **전체 일관성 정련 패스** — 모든 CQ 처리 후, 누적 온톨로지 전체에 대해 마지막 메타인지 정련을 1회 수행: 고립 클래스 연결, 이름 표준화, 명백한 중복 제거. 이 패스도 별도 단계 스냅샷으로 남긴다.
6. **산출** — 각 CQ의 초안/자기비평 후 스냅샷과 최종 정련 스냅샷을 남겨 "온톨로지가 비평을 거쳐 정련되는 과정"을 보여주고, `ontology.ttl`(OWL/Turtle), `ontology.json`(그래프), `steps.json`(단계별 스냅샷)을 쓴다.

> **CQbyCQ와의 대조**: CQbyCQ는 2단계(조각 변환 → 병합)로 맥락 없이 직진한다. Ontogenia는 CQ마다 **자기비평 패스가 1회 추가**되고, 끝에 **전체 정련 패스가 1회 더** 붙어 단계 수가 늘고 출력이 더 정합적이다.

## 4. 입력 / 출력
| 구분 | 파일 | 설명 |
|------|------|------|
| 입력 | `competency_questions.txt` | CQ 목록(한 줄 1개, `#` 주석·빈 줄 무시) |
| 출력 | `ontology.ttl` | OWL 온톨로지 (Turtle) |
| 출력 | `ontology.json` | Cytoscape nodes/edges (그래프 시각화용) |
| 출력 | `steps.json` | CQ별 초안·자기비평·최종 정련 스냅샷 — 단계별 재생용 |

## 5. LLM 백엔드
- 기본 `mock`: 키 없이 결정론적으로 동작(테스트 golden 고정). 초안 단계는 CQbyCQ와 같은 휴리스틱(대문자 명사→클래스, 관계 동사→객체 속성)으로 만들고, **자기비평 패스도 결정론 규칙으로 흉내** 낸다 — 예: 누적 그래프에 유사 이름 클래스가 있으면 병합, 단수/복수 정규화, 고립 클래스에 상위 클래스 부여.
- `gemini`/`anthropic`: 키가 있으면 실제 LLM이 초안 + 메타인지 자기비평 + 전체 정련을 수행해 더 정합적인 온톨로지를 만든다. 키 없으면 자동 MOCK 폴백.
- Ontogenia는 본질적으로 **CQ당 LLM 호출이 2회(초안+비평) + 마지막 정련 1회** 라서 CQbyCQ보다 토큰을 더 쓰지만 일관성이 높다.

## 6. 직접 해보기
1. `samples/competency_questions.txt` 를 수정하거나 새 CQ를 추가.
2. 사이트의 **Run** 버튼(또는 `python pipeline.py samples runs/out`) 실행.
3. 단계 슬라이더에서 각 CQ의 **초안 → 자기비평 후** 변화와, 마지막 **전체 정련** 단계까지 따라가며 비평이 그래프를 어떻게 정리하는지 확인.
